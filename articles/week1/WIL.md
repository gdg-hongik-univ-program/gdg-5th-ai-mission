### 데이터 전처리

데이터 전처리란 데이터 분석을 위해 수집한 데이터를 분석에 적합한 형태로 가공하는 과정을 말합니다. 데이터 전처리 과정이 중요한 이유는 데이터 분석에 사용되는 데이터의 질이 좋지 않다면 부정확한 결과와 분석과정에서 안정적이 빠른 작업 수행이 불가능하기에 데이터 전처리 과정은 데이터 분석 전 매우 중요한 과정이다.

데이터 전처리 방식은 매우 다양하고 데이터 전처리 과정도 길기에 텍스트 문장을 벡터화 시키는 TextVectorization에 대해서 정리해보겠습니다.

---

### TextVectorization이란?

자연어 형태인 텍스트 데이터를 컴퓨터가 이해할 수 있는 수치 벡터의 형태로 변형하는 데이터 전처리 과정을 말한다. 여기서 데이터 전처리란 데이터 분석을 위해 수집한 데이터를 분석에 적합한 형태로 가공하는 과정을 말한다. 불필요한 데이터를 제거하고 데이터들의 형태를 통일시킴으로서 데이터의 질을 향상 시킬 수 있다.

TextVectorization은 총 4단계로 이루어진다.

1. 표준화: 텍스트를 정제하는 과정으로 대문자를 소문자로 변화하거나 구두점등을 제거하여 단어들의 형태를 통일하는 과정이다.
2. 토큰화: 문장을 단어 단위로 쪼개는 과정이다.
3. 인덱싱: 각 토큰에 고유한 정수를 부여한 단어 사전을 만든다.
4. 벡터화: 각 토큰별 고유 정수를 사용해 문장을 숫자들의 배열로 바꾼다.

TextVectorizaion에는 다음과 같은 벡터화 방법들이 존재한다.

#### Integer Indexing

사람의 언어로 이루어진 문장을 컴퓨터에게 인식시키기 위해서 단어 사전을 만드는 과정으로 먼저 문장에서 단어들을 골라내어 단어의 집합을 만든다.
![](https://velog.velcdn.com/images/kyucho1009/post/b06949bf-9000-43f1-b7c4-ecc54545db54/image.png)
단어 집합에 있는 단어들에 각각의 고유 정수를 부여한다. 이때 각 단어에 대응되는 고유 정수는 많이 등장하는 단어일수록 낮은 고유 정수를 갖게 된다. 이때 문장들의 길이가 다를경우 padding을 이용해 0을 채워넣어 문장들의 길이를 맞춰준다.
![](https://velog.velcdn.com/images/kyucho1009/post/fd8165d8-e43a-4f05-9843-4e8cf4983aae/image.png)

#### One-Hot Encoding

각 단어를 사전의 크기만큼의 벡터로 표현하며 해당 단어의 인덱스만 1, 나머지는 0으로 표현한다.
![](https://velog.velcdn.com/images/kyucho1009/post/0e4417cd-b735-458f-905a-fe425f313de4/image.png)

#### Multi-Hot Encoding

문장 전체를 하나의 벡터로 표현하며 문장에 등장하는 모든 단어의 인덱스 위치를 1로 표시한다.

#### Count Encoding

Multi-Hot Encoding과 유사한 방식으로 단순히 등장 유무를 0과 1로 표현하는것이 아니라 등장횟수를 표기하는 방식이다.

#### TD-IDF(Term Frequency-Inverse Document Frequency)

단어의 빈도뿐만이 아니라 해당 단어가 전체 문서에서 얼마나 희귀한지를 고려하여 가중치를 부여하는 방식이다.

#### Vectorization 과정이 필요한 이유

위에서 다양한 TextVectorization 방법들을 알아봤는데 이러한 벡터화 과정이 왜 필요할까?
만약 벡터화 과정을 거치지 않는다면 컴퓨터는 문장들에 수학적 계산을 할 수 없으며 우리에게는 의미가 있는 문장일지 몰라도 컴퓨터에게는 이해할 수 없는 무의미한 문자열에 불과할뿐이다. 그리고 컴퓨터가 처리할 수 있는 고정된 형식의 데이터를 제공받지 않으면 모델이 더욱 안정적이고 빠른 처리를 할 수 없다.
그렇기에 위와 같은 다양한 벡터화 방법이 사용되어야한다.
